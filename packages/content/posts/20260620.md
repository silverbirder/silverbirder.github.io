個人ブログを、見つけて、読んで、シェアするアプリ、"こぶりー" を運営しています。  

https://kobliy.vercel.app

このアプリの肝の1つは、個人ブログのデータの数と思っています。  
どうすれば個人ブログをみつけることができるのか、自分の考えをまとめてみました。  

## キュレーションサイトから

私の個人ブログのアクセス分析を見ると、どこからアクセスされているかリファラを見ることができます。  
そのリファラを辿ると、キュレーションサイトだったりします。  
一番の数で言うと、ミンゲイインターネット さんですね。  

https://mingeiinter.net

他にも5件ほど用意していて、それらをクロールして個人ブログを収集しています。  

## 個人サイトから

Web は、ドキュメントとドキュメントを繋ぐリンクがあります。  
"こぶりー" で把握している個人ブログ記事のHTMLを解析し、リンクを集める方法を考えました。  
個人ブログが増えれば、個人ブログ記事も増えるため、ドンドン集まるのでは？と思ったのです。  

ただ実際の個人ブログ記事を1万件ほど探索して、1件ほどしか個人ブログを発見できなかったのです。  
費用対効果に合わなかったので、このやり方は見送りました。  
まあ確かに私の個人ブログでもリンクはしますが、個人ブログへリンクするのは稀だなと思うのでした。  

## GitHub から

主にIT業界で個人ブログを提供する人は、GitHub リポジトリで公開していることが多いです。  
そのため、GitHub のコード検索で、ブログっぽい単語で検索すると割とヒットします。  
例えば、github.ioのサイトや、リポジトリにリンクされている個人ブログサイトがあったりします。  
界隈が限定されるのは少し偏ってしまうのですが、まあヨシとしています。  

## ブログサービス から

note や アメブロ、はてなブログ などブログサービスを提供するサービスは数多くあります。  
これらにブログ記事を書かれている人はたくさんいます。  
会社として書いているのは個人ブログとして対象外と考えているので除外なのですが、  
純粋に面白そうな個人ブログは、"こぶりー"に取り入れています。  
ただ本当に山の様にあるので、ここの精査する方法は、より仕組み化が必要となります。  

## 終わりに

おかげさまで、個人ブログの数は2.7k、個人ブログ記事の数は137.3kもあります。  
"こぶりー" には、記事を探す方法を3つほど用意しています。  

- カレンダー等で自由に年月を移動して記事を探せる
- 記事の中からランダムに1つ読むことができる
- フィード説明文を元に記事検索することができる

より充実した "こぶりー" へ成熟させたいため、今後も頑張ってデータ集めをしようと思います。
