以下で個人ブログキュレーションのアプリを開発しています。

https://silverbirder.github.io/blog/contents/20260419/

ブログは、RSSやAtomのフィードを元にデータを整えています。

## 仕様

RSSやAtomの仕様は、以下のサイトで確認できます。

- RSS 2.0
  - [RSS 2.0 Specification (Current) - www.rssboard.org](https://www.rssboard.org/rss-specification)
- Atom
  - [ RFC 4287 - The Atom Syndication Format  - datatracker.ietf.org](https://datatracker.ietf.org/doc/html/rfc4287)

Feed 自体の必須項目は、以下の通りです。

- RSS 2.0
  - title
  - link
  - description
- Atom
  - title
  - id
  - updated

title は、フィードのタイトルです。  
link は、フィードをWeb URLです。  
Atom の場合、`link:rel="alternate"` が使われています。  

## linkが壊れてる

ブログキュレーションに最低でも link が必須なんですが、  
link が壊れてるフィードがあったりします。  
例えば、以下がありました。

- undefined, null などの文字
- localhost
- URLっぽいもの（文字の間に半角スペース）

link が相対パスのものは、絶対パスに置換できるので問題ないです。  
他には、link に末尾スラッシュへリダイレクトされるものもありました。  

## 壊れても大事にしたい

とはいえ、できれば大事なフィードなので、  
個別にフィードデータを抽出できる様に調整するようにしています。
