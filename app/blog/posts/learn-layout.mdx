## 何ができると楽しくなるか

私がCSSで自信が掴め始めたのは、CSSのレイアウトに関する技術が身についた頃からです。

レイアウトとは、要素をどのように配置するかということです。
CSSの根本である、ボックスレイアウトとフローレイアウトを理解すれば、レイアウトの基本がわかります。
レイアウトができるようになると、CSSできるようになってきた!というできる実感が湧いてきます。

そのため、本記事ではCSSレイアウトに重点を置いています。

他に、CSSに関して学ぶとしたら、他には色相やタイポグラフィ、角丸、影、アニメーション、色々ありますが、それらは大したことではありません。
やっていけば覚えますし、調べながらやれば十分です。

また、CSSは、いきなりできるようになるものではありません。
実際に業務を経験して、自信の血肉となるのです。
まずは、わかった気になるところまで足を踏み入れるのです。
そこから、『理屈はわかったので、多分できるはずだ...』という気持ちになれたら、本書のゴールです！

## 前提となる知識

レイアウトを学ぶ前に、前提となる知識をおさらいしておきましょう。
ちなみに、以降は 書字方向は横書きを前提としています。

### ボックスモデル

https://developer.mozilla.org/ja/docs/Web/CSS/CSS_box_model/Introduction_to_the_CSS_box_model

CSSは、以下を含めてボックスで表現されます。

- コンテンツボックス
- パディングボックス
- ボーダーボックス
- マージンボックス

### 通常フローとブロックとインライン

https://developer.mozilla.org/ja/docs/Web/CSS/CSS_display/Block_and_inline_layout_in_normal_flow

ボックスを配置する際に、横に並べるか縦に並べるかというのを考えます。
もちろん、絶対位置のような配置方法もありますが、まずは通常フローでの配置方法を考えます。

### フレックスボックス

https://developer.mozilla.org/ja/docs/Web/CSS/CSS_flexible_box_layout/Basic_concepts_of_flexbox

フレックスボックスは、アイテムを行・列に配置できます。
さらに、縮めたり伸ばしたりできるのが、特徴です。

デザインしたい要素を、フレックスボックスとして配置を考える場合、
トップダウンでHTML構造を考えます。
そこから、横に配置するか縦に配置するかを考えます。
それを何度も繰り返すことで、デザインできるようになります。

### グリッドレイアウト

https://developer.mozilla.org/ja/docs/Web/CSS/CSS_grid_layout/Basic_concepts_of_grid_layout

グリッドレイアウトは、二次元グリッドシステムをCSSで実現します。
フレックスボックスのように、縦横に配置するレイアウトも可能ですし、
グリッドエリアを指定して、複雑な2次元構造を表現もできるのが特徴です。
1次元ももちろん可能です。

### 重ね合わせコンテキスト

通常フローは、平面で考えます。
重ね合わせコンテキストは、z軸方向に要素を重ねることができる概念です。

https://developer.mozilla.org/ja/docs/Web/CSS/CSS_positioned_layout/Stacking_context

同じz軸の高さにある要素は、重なり順序を考慮する必要があります。z-indexですね。
逆に、高さが揃っていない要素同士は、比較することができません。比較は、同じ高さのものです。

重ね合わせコンテキストを生成するのは、positionが一番有名かと思います。
これを使う際は、position relativeしてabsoluteで配置することが多いです。

### ブラウザに計算させる

レイアウトを実装していくと、widthを100pxのように固定値でしてしまうことがあります。
これは、特定のブラウザサイズであればうまくいくかもしれません。
しかし、以下のような問題が発生するかもしれません。

- ブラウザの縦横の幅が変わると、レイアウトが崩れる
- 表示するデータが変動すると、レイアウトが崩れる

そのため、できる限りブラウザに計算させる方法をお勧めします。
具体的には、以下のような方法があります。

- 単位を%やdvw、cqwなど相対単位を積極的に使用する
  - max-widthで固定値で制限値をかける
- calc()を使用して、計算させる
  - calc(100% - 20px)のように、相対値と固定値を組み合わせる

そのため、固定で定義した際には、"ブラウザサイズやデータが変わったらどうなるのか？"を想像しましょう。

### デバッグ方法

CSSでなんでこうなるんだろう？とわからなくなることは、とてもよくあります。
そのような時は、ChromeなどのブラウザにあるDevToolsを使うと良いでしょう。

例えば、以下のデバッグができるので、実装時は重宝します。

- 要素のボックスモデルを確認する
  - コンテンツ、マージン、パディング、ボーダーのサイズを確認できます。
- flexboxやgridのレイアウトを可視化する
  - flexboxやgridのitemがどのように配置されているか確認できます。
- CSSのプロパティをその場で変えて、違いを確認する
  - align-itemsやjustify-contentなどのプロパティを変更して、どのように変わるか確認できます。

ref: https://developer.chrome.com/docs/devtools/css/reference

### 共通変数

サンプルページでは、スペースや色、フォントサイズをCSS変数で定義しています。
これは、CSSの値を設定する際に、`var(--spacing-xs)` のように使用します。
決まったスペースやフォントサイズを決めておくと、全体のデザインが統一されるためお勧めです。

### 全体セクション

ヒーローやヘッダー・フッターを除いたセクションは、以下のHTMLで囲むようにしています。

```html
<div class="container sections">
  <section></section>
  <section></section>
  <!-- 省略 -->
</div>
```

これは、以下のようなCSSになっています。
全体の最大横幅を定義しつつ、左右中央に寄せるようにします。
また、セクションごとに、gapによるスペースを開けるようにしています。

```css
.container {
  max-width: var(--breakpoint-desktop);
  margin-left: auto;
  margin-right: auto;
  padding: 0 var(--spacing-base);
}

.sections {
  display: grid;
  grid-template-columns: 1fr;
  gap: var(--spacing-lg);
  /* Footer向けのスペースです */
  margin-bottom: var(--spacing-lg);
}
```

3つに分けられそう。

HTML構造は、以下になりました。（正解は1つではありません）

```html
<section class="hero-cover">
  <main class="hero-container">
    <hgroup>
      <h1 class="hero-title">CSS AI</h1>
      <p class="hero-subtitle">AI駆動CSSスタイリング<wbr />プラットフォーム</p>
    </hgroup>
    <div>
      <p class="hero-description">
        次世代AIがあなたのCSSを自動最適化。
      </p>
      <p class="hero-description">
        自然言語でスタイルを指定するだけで、美しいCSSコードを瞬時に生成します。
      </p>
    </div>
    <div class="cluster">
      <button class="btn btn-primary">今すぐ始める</button>
      <button class="btn btn-outline">詳細を見る</button>
    </div>
  </main>
</section>
```

CSSでは、以下を実現したいです。

- 画面全体を広げる
- コンテンツの横をある程度に限定したい
  - 横に広げすぎるより、横幅を制限したい
- 左右上下中央に配置している
- 3つ(タイトル、説明、ボタン)を縦に並べる

これも想像してみましょう。
特に、左右上下中央は、さまざまな方法があります。
手っ取り早い方法は、grid の place-items: centers です。
ただ、flexboxを使ったりもできますので、お好みです。

CSSは、以下のようになりました。

```css
/* Hero Section */
.hero-cover {
  /* 左右上下中央の配置します。 */
  display: grid;
  place-items: center;
  /* ビューポートの最大まで表示します */
  min-height: 100dvh;
}

.hero-container {
  display: grid;
  /* 1列で、横幅いっぱいに表示します */
  grid-template-columns: 1fr;
  justify-items: center;
  gap: var(--spacing-lg);
  /* コンテンツの横を制限させる */
  max-width: var(--breakpoint-desktop);
  padding: 0 var(--spacing-base);
}

.hero-title {
  font-size: var(--font-size-4l);
  font-weight: var(--font-weight-bold);
  text-align: center;
}

.hero-subtitle {
  font-size: var(--font-size-lg);
  font-weight: var(--font-weight-normal);
  text-align: center;
  word-break: keep-all;
}

.hero-description {
  font-size: var(--font-size-base);
  font-weight: var(--font-weight-normal);
}

.cluster {
  /* ボタンを単純に横に並べる */
  display: flex;
  /* ボタンが増えると折り返す */
  flex-wrap: wrap;
  gap: var(--spacing-sm);
  justify-content: center;
  align-items: center;
}
```

ちなみに、モバイルのように横幅が狭いデバイスでは、
wbr要素とword-breakを使って、テキストの折り返しを制御することができますので、ご活用ください。
